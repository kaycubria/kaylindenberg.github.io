Get Git

By: Kay Lindenberg & Christopher Selden

Git is a version control system that lets you track and store code and changes.

Git Changes

Git makes it easy to check edits made to code and adding changes

$ git status => Shows files that were added/removed/changed
$ git add(rm) [file] => Stages code for commit
$ git reset [file] => Unstages the file
$ git commit -m "[commit message about changes]" => Adds note about the changes
Git Configuration

Git lets you configure your information for your repositories

$ git config --global user.name "[name]" => Automatically sets your username when you commit code
$ git config --global user.email "[email address]" => Automatically sets your email when you commit code
Git Branches

How to move through, update and delete branches

$ git branch => Returns the branches in the respository
$ git branch [branch-name] => Creates a new branch
$ git checkout [branch-name] => Switches which branch you are working on
$ git branch -d [branch-name] => Deletes the branch
Git Repositories

Create or clone a repository from a URL

$ git init [project-name] => creates a new repository
$ git clone [url] => Clones a repository from a GitHub URL
Git Log

Look at the history of changes

$ git log => Lists history of changes for the current branch
$ git diff [first-branch]...[second-branch] => Shows differences between the two branches
Git Sync

Update your local repository

$ git fetch [bookmark] => Takes all the changed data from the online version of the repository
$ git merge [bookmark]/[branch] => Updates the local repository with the downloaded changes
$ git push [alias] [branch] => Uploads local changes to the online repository
$ git pull => Downloads online updates and incorporates changes
